# Codex Rules

- いきなり巨大実装しない。
- 小さい差分で進める。
- 既存ファイルを壊さない。
- 外部 API 連携は明示指示があるまでやらない。
- Google Calendar / Gmail / Notion の本連携は勝手に実装しない。
- API キーやトークンをコードに直書きしない。
- `.env` をコミットしない。
- 変更後は可能な範囲で test / typecheck / lint を実行する。
- 作業後に vault を更新する。
- 破壊的変更の前に確認する。
- README と vault の内容が古くなったら更新する。
- 実装できなかったことは隠さず報告する。
